# ウマ育成・着せ替えゲーム 開発仕様書

このリポジトリで作るものと、守ってほしいルールをすべてここに書く。
実装を始める前に、必ず「未確定事項」の章を読み、判断が必要な箇所は先に質問すること。

---

## 1. ゲーム概要

草むらをタップするとウマが現れ、そのウマから**パーツ（色 or 装飾）**が手に入る。
集めたパーツを組み合わせて自分だけのウマを作り、コレクションしていく着せ替えゲーム。

- プラットフォーム: **Webアプリ（スマホ縦画面ファースト）**
- データ保存: **ローカル保存のみ**（サーバーなし。まずは1人用として完成させる）
- ウマの見た目: **SVGレイヤー合成**でリアルタイム描画（画像ファイルは使わない）

---

## 2. 技術スタック（この構成で作る）

| 項目 | 指定 |

| フレームワーク | React 18 + TypeScript |
| ビルド | Vite |

外部の課金・広告・ネットワーク通信は一切入れない。

## 3. ウマの描画モデル（最重要）

ウマは**1枚のベースSVG + 3つの色変数 + 装飾レイヤー**で構成される。

### 3.1 ベース

`assets/horse_base.svg` を使う。横向き・右向き・輪郭線 `#2b2118` / 太さ8。viewBox は `0 0 520 520`。
ベース内の塗りは CSS変数で制御する:

- `--body` … 胴・首・頭・脚・耳
- `--mane` … たてがみ・しっぽ
- `--hoof` … ひづめ4つ
- 白目・鼻・お腹・マズルの白は固定色（変更しない）

### 3.2 装飾レイヤー

装飾は**スロット制**。1スロットにつき装備できるのは1個まで。

| スロット | 内容 | 描画順（後 = 手前） |
|---|---|---|
| `back` | つばさ / リュック / くら / アーマー / ジェットパック | ベースの**後**に描画 |
| `tail` | リボン / ほのお / こおり / きらめき | ベースの後 |
| `head` | ハット / 角 / かぶと / ヘルム / アンテナ | ベースの後 |
| `face` | バンダナ / ゴーグル / バイザー | 最前面 |

`src/data/parts.json` に全パーツが定義済み。**このJSONを唯一の正とする**（ハードコード禁止）。
装飾のSVGスニペットは `svg` フィールドにそのまま入っているので、ベースと同じ座標系（0 0 520 520）で重ねるだけでよい。

### 3.3 レンダラー

`<HorseView horse={Horse} size={number} />` という単一コンポーネントを作る。
このコンポーネントは**全画面で使い回す**（草むら・図鑑・作成・一覧すべて）。中で `<svg viewBox="0 0 520 520">` を組み立て、
`style={{ '--body': ..., '--mane': ..., '--hoof': ... }}` を渡し、装備中の装飾スロットを描画順に挿入する。

---

## 4. パーツ一覧（parts.json）

- 色: body 12種 / mane 9種 / hoof 6種 = **27種**
- 装飾: head 9 / face 3 / back 5 / tail 4 = **21種**（※JSONの実数を正とする）
- 合計 **48種** が図鑑のマス数になる

各パーツは `rarity` を持つ: `N` / `R` / `SR`

---

## 5. 草むら（メイン画面）の仕様

### 5.1 出現枠のリセット

- 1日2回、**12:00** と **00:00**（端末ローカル時刻）に草むらがリセットされ、ウマを出せるようになる
- リセットのたびに出現枠は **1** に戻る（枠は貯まらない。上限1）
- 枠がある状態で草むらをタップ → ウマが現れる → パーツ獲得 → 枠を消費
- 枠がない間は、次のリセットまでのカウントダウンを表示する

実装方針: 枠の有無は「最後にウマを出した時刻」と「直近のリセット時刻」の比較で判定する。
アプリを開いていない間の経過も正しく扱えるよう、**タイマーではなく時刻比較**で実装すること。
`lastSpawnAt < 直近のリセット時刻` なら枠あり。日跨ぎ・アプリ再起動・長期放置でも破綻しないこと。

### 5.2 演出

タップ → 草がゆれる → ウマがぴょこっと飛び出す → 獲得パーツがカードでめくれる、という一連の流れ。
過剰なエフェクトは不要。**草のゆれ → 出現 → 獲得** の3ステップだけを気持ちよく作る。
`prefers-reduced-motion` を尊重すること。

### 5.3 獲得抽選

1回の出現で、**色と装飾をまとめた全パーツプール（48種）から N 個**を獲得する。

| 獲得個数 | 確率 |
|---|---|
| 1個 | 50% |
| 2個 | 40% |
| 3個 | 7% |
| 4個 | 3% |

個数を決めたあと、その個数だけパーツを抽選する。パーツ単体の抽選はレアリティ重み付き:

| レアリティ | 重み |
|---|---|
| N | 70 |
| R | 25 |
| SR | 5 |

- 同一抽選内で**同じパーツは重複させない**（引き直す）
- すでに所持済みのパーツを引いた場合は「かぶり」として扱い、`duplicates` カウントを +1 する
  （かぶりの用途は将来のレース／投票機能で使うため、カウントだけ持っておく）
- 抽選ロジックは `src/logic/gacha.ts` に純粋関数として切り出し、Vitestでテストする
  （個数分布・重み・重複なしを検証）

## 6. 画面構成

タブ or ボトムナビで以下を切り替える。

### (1) 草むら（`/`）— メイン
上記5章の内容。次のリセットまでの残り時間、所持ウマ数（`n/10`）を常時表示。

### (2) マイウマ（`/stable`）— 所持ウマ一覧
作成済みのウマを一覧表示（最大10体）。タップで拡大表示、名前の変更、削除ができる。
「削除すると戻せない」ことを明示し、確認ダイアログを出す。

### (3) つくる（`/create`）— ウマ作成
所持しているパーツだけを選んでウマを組む画面。

- body / mane / hoof の色を1つずつ選ぶ（未所持の色は選べない。グレーアウト表示）
- head / face / back / tail の装飾を**それぞれ0個または1個**選ぶ（装飾なしも成立する）
- プレビューはリアルタイム更新（`<HorseView>` を使う）
- 名前をつけて保存 → マイウマに追加
- **所持上限は10体**。上限時は保存ボタンを無効化し、「マイウマで1体消すと作れます」と案内する
- 既存のウマの編集もこの画面で行う（作り直しではなく上書き）

### (4) 図鑑（`/collection`）— コレクション
**パーツ単位**で並べる（ウマ単位ではない）。セクションは以下の順:

1. ボディカラー（12）
2. たてがみカラー（9）
3. ひづめカラー（6）
4. 頭（9）/ 顔（3）/ 背中（5）/ しっぽ（4）

- 未所持は**シルエット表示**（色は伏せて `?` マーク、装飾は輪郭のみ）
- 所持済みは実物 + 名前 + レアリティ + 入手個数
- 上部に達成率（`所持数 / 48`）を出す

### (5) レース（`/race`）— **後日実装**
今回はプレースホルダー画面のみ作る。「準備中」とだけ表示し、ナビからは到達できるようにしておく。
将来ここで所持ウマを走らせる予定なので、`Horse` 型には拡張余地を残すこと（ただし今はステータス値を作らない）。

### (6) 人気投票（`/vote`）— **後日実装**
同上、プレースホルダーのみ。将来オンライン化する可能性があるため、
ローカルデータ構造は**ウマ1体を単体でシリアライズできる形**にしておくこと。

## 7. データモデル

```ts
type Rarity = 'N' | 'R' | 'SR';
type ColorSlot = 'body' | 'mane' | 'hoof';
type DecoSlot  = 'head' | 'face' | 'back' | 'tail';

type Horse = {
  id: string;
  name: string;
  colors: Record<ColorSlot, string>;          // parts.json の色パーツ id
  decos: Partial<Record<DecoSlot, string>>;   // parts.json の装飾パーツ id（未装備はキーごと無し）
  createdAt: number;
};

type SaveData = {
  version: 1;
  owned: Record<string, number>;   // パーツ id -> 入手個数（1以上なら所持）
  horses: Horse[];                 // 最大10
  lastSpawnAt: number | null;      // 最後に草むらからウマを出した時刻(ms)
};
```

保存は状態変化のたびに localStorage へ。読み込み時に `version` を見て、壊れていたら初期化する。

## 8. 見た目の方向性

サンプルにした恐竜ゲームと同系統の、**手描き風・厚い輪郭線・くすんだ土色のUI**。

- 背景・パネル: 生成りの紙／なめし革のような色（`#c8a06a` 系のパネル、`#8c6a41` の内側枠線）
- 文字色: `#3a2c1c`
- ボタンは角丸の立体風。決定は青、キャンセルは赤（サンプル画像に準拠）
- フォントは丸ゴシック系を優先（日本語が主言語）
- 情報の密度は低く、1画面1目的。スマホ片手で完結すること

派手なグラデーションやガラス風エフェクトは使わない。

## 10. 未確定事項（実装前に確認する）

以下は仕様が確定していない。**勝手に決めず、着手前にまとめて質問すること。**

1. 獲得個数の表記ゆれ: 元の指示は「1〜3個」だが、確率表は4個まで存在した（50/40/7/3 = 100%）。
   → 本仕様書では**最大4個**として記述している。1〜3個に変更する場合は確率の再配分が必要。
2. 所持上限「10体」は**ウマの体数**という解釈で書いている。
3. かぶり（重複パーツ）の使い道。現状はカウントのみ。
4. レース／人気投票の中身（勝敗の決め方、投票の対象と周期）。
